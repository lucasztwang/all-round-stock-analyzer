# A-Stock Analyzer

## 🧭 意图决策树（必须优先执行）

```
用户说：
├─ "选股/筛选/找票/推荐/有什么好票/帮我挑" → 🤖 选股路径（下文"路径A"）
├─ "分析/怎么看/看看/怎么样/这只股/帮我看" → 🔍 股票分析路径（下文"路径B"）
├─ "短线/尾盘/一夜持股/隔夜/尾盘选股/打短/超短" → ⏰ 短线尾盘选股（下文"路径C"）
└─ 无法判断 → 🔍 默认股票分析路径 + 询问"您是想选股还是分析某只具体的股票？"
```

---

## 🤖 路径A：选股路径

适用于用户想从候选池中筛选值得关注的股票。

### 流程（6步）

```
1. 确定候选池 → 用户指定板块/范围，或询问用户偏好
2. 数据获取   → 逐只获取实时行情
3. 四步选股   → 排雷→估值→精选→量价，逐只打分排序
4. 进阶信号   → 趋势动能+小市值+财报验证
5. 仓位建议   → 三层仓位初始分配
6. 买卖点初判 → 对比各候选股买入信号强弱
→ 输出：多只对比表格 + 雷达图 + 仓位建议
```

### 涉及模块

| 顺序 | 模块 | 用途 |
|------|------|------|
| 1 | `fetch_data` | 获取候选池数据和 K 线 |
| 2 | `stock_selection` → `full_selection_analysis()` | 四步选股+进阶信号 |
| 3 | `position_management` → `three_layer_position_plan()` | 三层仓位初始分配 |
| 4 | `buy_sell_point` → `batch_buy_sell_compare()` | 多只买卖点横向对比 |
| 5 | `charts` → `plot_selection_radar()` | 选股雷达图 |
| 6 | 直接输出 | 紧凑表格+雷达图（不生成完整报告） |

## 🔍 路径B：股票分析路径

适用于用户想深度分析某只具体股票。

### 流程（7步）

```
0. 询问持仓   → 成本价？仓位？持有时间？（每次必问）
1. 数据获取   → 行情+K线+财务
2. 基本盘分析 → 估值/成长/盈利
3. 产业逻辑   → WebFetch 东方财富F10 + WebSearch 公告/研报
4. 技术+买卖点 → 趋势/指标/信号 + 买卖点综合研判
5. 缠论分析   → 分型/笔/中枢/买卖点
6. 持仓调仓   → 止盈止损+仓位调整（结合持仓信息）
7. 生成报告   → 深度报告+6图+调仓建议
```

### 步骤0：询问持仓（必须执行）

**每次分析前必须询问用户**：

> 请确认以下持仓信息：
> 1. 您目前是否持有这只股票？
> 2. 如果持有，成本价是多少？
> 3. 目前仓位比例？
> 4. 持有时间多长？
> 5. 有什么产业逻辑或操作思路想补充？

如果用户不持有，则成本价和仓位留空，止盈止损模块给出中性分析。

### 步骤3：产业逻辑分析

使用 `industry_logic.py` 的辅助函数生成 F10 URL + 搜索词：

1. **调用 `f10_urls(code)`** 获取东方财富 F10 页面 URL
2. **使用 WebFetch** 获取 F10 财务摘要页面，提取 ROE/毛利率/营收增速等
3. **使用 WebSearch** 搜索最新公告、研报、产业动态
4. **调用 `analyze_industry_logic()`** 综合输出产业逻辑评分和分析

### 涉及模块

| 顺序 | 模块 | 用途 |
|------|------|------|
| 0 | — | 询问持仓信息 |
| 1 | `fetch_data` | 获取行情+K线+财务 |
| 2 | `fundamental` | 估值/成长/评分 |
| 3 | `industry_logic` + WebFetch + WebSearch | F10财报+公告+研报 |
| 4 | `technical` + `buy_sell_point` | 技术指标+买卖点研判 |
| 5 | `chan_theory` | 缠论分析 |
| 6 | `stop_profit` + `position_management` | 止盈止损+调仓建议 |
| 7 | `charts` + `report` | 图表+报告生成 |

## ⏰ 路径C：短线尾盘选股（一夜持股法）

适用于短线交易者在下午2:30后筛选隔夜持股标的，次日开盘30分钟内离场。

### 核心策略

- **操作时间**：下午2:30~2:55（尾盘最后30分钟确认）
- **持仓时间**：隔夜（今日尾盘买入，次日开盘30分钟内卖出）
- **仓位**：单票3%~5%，总仓位不超过20%（快进快出）
- **核心纪律**：红盘勿贪，绿盘别扛；无符合标的时空仓等待

### 8条件筛选逻辑

```
① 尾盘时间 → 下午2:30后操作（非交易时段为预筛模式）
② 涨幅3%-5% → 不<3%太弱，不>5%太高
③ 量比>1 → 放量确认 ⭐回测最强因子（<1.0胜率仅26%, ≥1.5胜率45%）
④ 换手率5%-10% → 活跃但非异常放量
⑤ 流通市值30-150亿 → 小盘弹性好
⑥ 成交量阶梯放大 → 逐步放量非脉冲
⑦ 盘中强于大盘 → 全天走势强于指数
⑧ 尾盘高位回踩 → 创盘中新高后回踩不破均线
```

> 回测验证（150只×10天,114笔模拟,5分钟K线）：≥6.5/8分胜率~50%；量比是最强预测因子。

### 流程（5步）

```
0. 时间检查   → 判断是否在交易时段 + 是否已过2:30
1. 全市场粗筛 → 涨跌幅3%-5% + 流通市值30-150亿（腾讯API批量）
2. 精细筛选   → 逐只获取实时数据，8条件逐项打分
3. 排序输出   → 8条件打分表 + 前1-3只推荐 + 明日操作提醒
4. 操作纪律   → 强制提醒：次日开盘30分钟离场、止损线-3%
```

### 涉及模块

| 顺序 | 模块 | 用途 |
|------|------|------|
| 0 | `fetch_data` → `get_market_status()` | 时间检查（交易时段判断） |
| 1 | `fetch_data` → `get_market_snapshot()` | 全市场批量行情快照（~5500只） |
| 1 | `fetch_data` → `get_index_quote()` | 大盘指数实时行情 |
| 2 | `tail_market_selection` → `run_tail_market_selection()` | 8条件筛选+格式化输出（一键全流程） |

### 步骤0：时间检查

调用 `get_market_status()` 判断当前市场状态：

- **尾盘时段（14:30-15:00）**：正式操作模式，数据实时性最高
- **非尾盘时段**：预筛模式，输出带 `[预筛模式]` 标记，提示下次可操作时间

### 步骤1-3：自动执行

调用 `run_tail_market_selection()` 一键执行全流程：
1. 获取全市场行情快照（~5500只A股实时数据）
2. 粗筛（涨幅3%-5% + 流通市值30-150亿）
3. 精细筛选（8条件逐项打分，按总分排序取前3名）

### 步骤4：输出纪律提醒

必须在输出中包含强制操作纪律表格（持仓时间、止损线、仓位上限、空仓纪律、绿盘规则）。

## ⚠️ 重要提醒

- **路径区分**：看到用户消息后先判断意图，走对路径
- **持仓必问**：分析路径每次必问持仓信息
- **产业逻辑**：分析路径必须使用 WebFetch + WebSearch 获取 F10 数据
- **数据源**：腾讯API为实时行情+K线，东方财富为F10补充
- **免责声明**：报告中必须包含"不构成投资建议"
